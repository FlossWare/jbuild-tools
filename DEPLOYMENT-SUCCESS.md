## 🚀 What's Now Automated Across All Projects

### 1. Continuous Quality Monitoring
- ✅ Runs on **every push** to main/develop branches
- ✅ Runs on **every pull request**
- ✅ **Daily security scans** at 2 AM UTC

### 2. Automatic Issue Creation
GitHub issues are automatically created for:
- 📊 **Code coverage drops** below 93% instruction / 86% branch
- 🐛 **SpotBugs violations** (any bugs found)
- 📝 **PMD violations** (code quality issues)
- ✓ **Checkstyle errors** (style violations)
- 🔒 **Security vulnerabilities** (OWASP critical/high)

### 3. Pull Request Quality Gates
Every PR automatically gets:
- 📊 **Quality metrics table** in comments
- ✅/❌ **Pass/fail status** for each tool
- 🚫 **Merge blocking** if quality gates fail
- 📎 **Links to detailed reports** (artifacts)

### 4. Smart Issue Management
- **De-duplication**: One issue per category (prevents spam)
- **Auto-labeling**: `quality-gate`, `automated`, tool-specific labels
- **Fix instructions**: Every issue includes step-by-step guide
- **Resource links**: Documentation and templates included

## 🛠️ Customization Options

Each project can customize its workflow:

### Adjust Coverage Thresholds

Edit `.github/workflows/quality-gate.yml`:

```yaml
- name: Create issue for coverage drop
  if: |
    steps.jacoco.outputs.instruction_coverage < '93%'  # Change here
```

### Change Schedule

```yaml
schedule:
  - cron: '0 2 * * *'  # Daily at 2 AM UTC - adjust as needed
```

### Disable Specific Tools

```yaml
- name: Run Maven verify
  run: mvn clean verify -Dspotbugs.skip=true  # Skip SpotBugs
```

### Modify Issue Templates

Edit the `body:` section of each issue creation step.
